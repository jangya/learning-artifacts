What is WebMCP?

A browser-native bridge between an AI agent and your existing web app.

A page exposes named JavaScript functions or annotated forms as structured tools. The browser lets an agent discover and call those tools while the user stays in the same visible, authenticated page.

IntentUser request
→
ReasoningBrowser AI agent
→
Web layerWebMCP tools
→
Your stackUI · JavaScript · APIs

The browser mediates the call; WebMCP reuses the page’s current state and application logic.

Without WebMCPObserve → guess → click → observe again

The agent may repeatedly consume screenshots, page text, or DOM context to understand each step.

With WebMCPDiscover schema → call tool → receive result

Compact schemas and structured results can reduce unnecessary context—and therefore token use—without hiding the UI.

◫Less context

Send purposeful tool arguments instead of repeatedly interpreting the entire interface.

✓More reliable

Tools connect to application logic, so visual redesigns are less likely to break agents.

◎Shared control

Actions update the same visible page, preserving session state and human review.

WebMCP is an emerging proposal inspired by MCP. It complements backend MCP integrations rather than replacing them.
